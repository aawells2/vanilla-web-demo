Vanilla Website

HTML

HTML, or HyperText Markup language, defines the structure of websites. There are a couple of core concepts that you will need to understand as a beginner.

  1. Semantic Elements
  2. Block vs Inline Elements
  3. Forms
  4. Aria Tags

Semantic Elements

According to google, semantics are "the branch of linguistics and logic concerned with meaning." 1 For example, angry and furious technically mean the same thing, however, they carry different connotations. The web works in a similar way.

For example, below are examples of bolded words:

  • Hello World
  • Hello World
Although they look the exact same, they use different html elements to mark them up. To be semantic, in html, is to make it easier for screen readers and other assistive technologies to understand the content. It is therefore very important to make sure that our html is semantic

Block vs Inline elements

Block elements, such as the paragraph tag, take up the entire space in its parent container. Inline Elements only take up the required space. This is important to note when you are styling your website - because it may do something you do not expect.

Forms

Forms are a very common sructure in all websites. Although it may look simple, it is not. Forms can be tricky, especially in later chapters, as we explore concepts such as validation.

Aria Tags

Aria (Accessible Rich Internet Applications) are attributes which define more complicated structures. For example, structures like accordions and carousels are not HTML elements. These structures make it tricky for screen readers to interact with them properly.
